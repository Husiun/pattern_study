设计模式之适配器模式
适配器模式是一个扩展模式，软件设计开始阶段是不会考虑的，只有在需要扩展系统的时候会考虑到；
适配器模式涉及到三个角色：
1.target 适配目标  期待得到的接口
2.adptee 适配源  需要适配的接口
3.adpter 适配器
适配器分为对象适配器和类适配器；
    适配器把原本不能一起工作的两个接口通过适配到一起，简单点理解就是电源转化器，插座转换器一类的东西
    类适配器通过继承适配类转换到target类，对象适配器通过持有适配接口转换为target；
    网上还有分的仔细的还有一种适配器叫缺省适配器default adapter;缺省适配器是应用中的一个特例，简单来说就是一个类
    需要接口中定义的某个方法，但是不需要全部的方法，这时直接实现接口是不能通过java编译的，所以采用中间类的情况，这个中间类
    就相当于一个适配器；
