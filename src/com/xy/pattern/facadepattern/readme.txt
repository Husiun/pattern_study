设计模式之门面模式：
    门面模式就是一个外观模式，它将一些细节你不必需要知道的东西隐藏起来，利用门面(Facade)进行封装，
    暴露给你的就是一个很简单的功能的调用。
    生活中很多实际的例子可以去解释这个模式：
    1.去酒店开房，大家都知道去去开房需要办理入住手续，需要办门禁卡，严一点的甚至需要去核实你的身份等等一系列工作，
    想想都挺复杂，但是现实却没这么做，我们只需要到前台去登记一下什么都好了，那么这个前台的功能呢就相当于酒店的门面，他帮你
    完成了一系列繁琐的事情,也就是程序中说的封装。
    以上是比较详细的说明，像我们去买房买东西都是一样的道理，只需要跟这些的门面打交道而不需要关心细节的问题


    门面模式涉及到三个角色：
    1.门面角色  模式的核心角色，需要熟悉子系统的功能，将对应的功能封装起来
    2.子系统角色  具体的子系统，有具体的功能实现在里面
    3.客户角色   通过调用facade来完成需要的功能
